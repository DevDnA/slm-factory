slm-factory + AutoRAG

도메인 문서로 특화 소형 언어모델(SLM)을 자동 생성하고, RAG 시스템과 결합하여 할루시네이션 없는 정확한 답변을 제공합니다.

왜 SLM + RAG인가

범용 LLM만으로는 도메인 정확도와 비용 효율성을 동시에 달성할 수 없습니다.

비용 절감

대형 모델 API 호출 비용 대비 10배 이상 저렴한 로컬 추론. GPU 서버 한 대로 전사 서비스 가능.

데이터 보안

민감한 사내 문서가 외부 API로 전송되지 않습니다. 온프레미스 완전 격리 운영.

도메인 정확도

SLM이 도메인 지식을 내재화하고, RAG가 최신 문서를 검색하여 할루시네이션을 근본 차단.

slm-factory

도메인 문서를 넣으면 특화된 소형 언어모델이 자동으로 만들어집니다.

자동화된 파이프라인

문서 투입부터 모델 배포까지 한 번의 명령으로 완료. wizard가 단계별로 안내합니다.

자동 진화

새 문서가 추가되면 tool evolve 한 번으로 증분 학습, 품질 게이트, 버전된 배포까지 자동 실행.

다중 형식 지원

PDF, HWPX(한글), HTML, DOCX, TXT 등 실무 문서 형식을 그대로 투입할 수 있습니다.

SLM + RAG 통합 아키텍처

slm-factory가 만든 SLM과 AutoRAG의 검색 파이프라인을 결합하여 최적의 답변 품질을 달성합니다.

핵심 연동 포인트 — slm-factory의 export.ollama.model_name이 AutoRAG 설정의 model: 값이 됩니다. Ollama가 두 시스템을 연결하는 브릿지 역할을 합니다. 별도 인프라 구축 없이 로컬에서 즉시 통합할 수 있습니다.

도메인 문서가 축적되는 모든 분야에 적용할 수 있습니다.

1

장애 대응 AI

장애 매뉴얼, 사고 보고서, 복구 절차서를 학습하여 장애 감지 즉시 대응 가이드를 자동 추천. 새로운 사례 추가 시 자동 진화.

2

사내 규정 Q&A

인사 규정, 보안 지침, 업무 매뉴얼을 학습하여 직원 질문에 정확한 근거와 함께 답변. 규정 변경 시 RAG에 즉시 반영.

3

제품 기술 지원

제품 매뉴얼과 FAQ를 학습하여 고객 문의에 정확한 기술 답변 제공. CS 인력 부담 감소.

4

공공/금융 문서 분석

법령, 고시, 내부 지침을 학습하여 담당자가 관련 조항을 빠르게 조회. 데이터가 외부로 유출되지 않는 온프레미스 운영.

도입부터 운영까지

3단계로 도입하고, 자동 진화로 지속 운영합니다.

Step 1. 초기 구축

도메인 문서 투입 → slm-factory wizard 실행 → 특화 SLM 자동 생성. 약 1~2시간이면 첫 모델 완성.

Step 2. RAG 연동

AutoRAG로 최적 검색 파이프라인 자동 탐색. SLM을 Generator로 연결하여 통합 시스템 구성.

Step 3. 자동 진화

새 문서 추가 시 tool evolve 한 번으로 증분 학습 → 품질 게이트 → 버전된 모델 자동 배포.

지속적 진화 — 모델이 한 번 만들어지고 끝이 아닙니다. 새로운 문서가 추가될 때마다 자동으로 재학습하여 지식이 계속 확장됩니다. 기존 QA는 유지하면서 새 문서만 처리하는 증분 학습 방식으로 효율적으로 운영합니다.

검증된 오픈소스 기반으로 구축됩니다.
