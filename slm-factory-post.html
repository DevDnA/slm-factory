slm-factory + AutoRAG

도메인 문서로 특화 SLM을 자동 생성하고, RAG와 결합하여 할루시네이션 없는 답변을 제공합니다

범용 LLM, 왜 그대로 쓰면 안 되는가

ChatGPT, Gemini 등 범용 모델을 업무에 바로 적용하면 다음과 같은 문제가 발생합니다.

  • 할루시네이션그럴듯하지만 틀린 답변을 자신있게 생성
  • 도메인 지식 부족사내 규정, 전문 용어를 알지 못함
  • 보안 우려외부 API로 내부 데이터가 유출될 위험

해결: slm-factory + AutoRAG

slm-factory는 도메인 문서(PDF, HWPX, HTML 등)를 입력하면 Teacher-Student 지식 증류 방식으로 도메인 특화 소형 언어모델(SLM)을 자동 생성합니다. 여기에 AutoRAG를 결합하면, SLM이 알고 있는 도메인 지식 위에 실시간 문서 검색까지 더해져 출처가 명확한 정확한 답변이 가능합니다.

slm-factory로 생성한 SLM을 Ollama에 배포하고, AutoRAG가 이를 호출하여 검색 증강 답변을 생성합니다

slm-factory 주요 기능

  • 다중 형식 파싱 (PDF, HWPX, HTML, DOCX)
  • Teacher-Student 지식 증류로 QA 자동 생성
  • LLM 기반 품질 평가 + 검증
  • LoRA 파인튜닝 (GPU/MPS/CPU)
  • 원클릭 Ollama 배포
  • 자동 진화 (증분 학습 + 품질 게이트)
