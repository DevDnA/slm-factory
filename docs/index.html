slm-factory 문서

도메인 문서로부터 특화된 소형 언어모델(SLM)을 자동 생성하는 Teacher-Student 지식 증류 프레임워크

문서 홈

개요

slm-factory는 도메인 문서(PDF, HWPX, HTML, TXT, DOCX)에서 소형 언어모델(SLM)을 자동으로 생성합니다. 대형 언어모델(Teacher)이 문서를 읽고 QA 데이터를 생성하면, 소형 모델(Student)이 이를 학습하여 특정 분야에 특화된 경량 모델을 만듭니다.

한 줄 요약 — 도메인 문서를 넣으면, 파싱 → QA 생성 → 검증 → LoRA 학습 → Ollama 배포까지 한 번의 명령으로 자동화합니다.

빠른 시작

1. 설치

git clone https://github.com/DevDnA/slm-factory.git
cd slm-factory
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[all]"

2. Ollama 준비

ollama serve              # 별도 터미널
ollama pull qwen3:8b      # Teacher 모델 다운로드

3. 프로젝트 생성 및 실행

# 프로젝트 초기화
slm-factory init my-project

# 학습할 문서 추가
cp /path/to/documents/*.pdf my-project/documents/

# wizard로 전체 파이프라인 실행 (권장)
slm-factory tool wizard --config my-project/project.yaml

4. 모델 테스트

cd my-project/output/merged_model
ollama create my-project-model -f Modelfile
ollama run my-project-model

wizard가 문서 선택부터 파싱, QA 생성, 검증, 학습, 배포까지 단계별로 안내합니다. 상세한 진행 방법은 사용 가이드를 참조하십시오.

기술 스택

카테고리 주요 패키지 역할
Core typer, pydantic, pyyaml, rich, httpx CLI, 설정 관리, 출력 포맷팅
Parsing pymupdf, beautifulsoup4, lxml, python-docx 문서 파싱 (PDF, HTML, DOCX)
ML/Training torch, transformers, peft, trl, accelerate LoRA 파인튜닝
Evaluation evaluate, rouge-score, nltk 모델 평가 (BLEU/ROUGE)
TUI textual 대시보드, QA 리뷰 인터페이스

시스템 요구사항

항목 최소 요구사항 권장 사양
Python 3.11 이상 3.11 또는 3.12
GPU CPU 가능 (매우 느림) NVIDIA CUDA (VRAM 8GB+) 또는 Apple Silicon
Ollama 1.0 이상 최신 버전
디스크 5GB 이상 20GB 이상

Apple Silicon 참고 — macOS에서는 BitsAndBytes 양자화를 사용할 수 없지만, Unified Memory 구조 덕분에 시스템 RAM 전체를 GPU가 공유하므로 양자화 없이도 비교적 큰 모델을 로드할 수 있습니다.
